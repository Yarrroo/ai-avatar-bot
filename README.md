# AI Avatar Chat Bot

Telegram-бот с ИИ-аватарами, двухуровневой системой памяти и стримингом ответов.

## Возможности

- **3 уникальных аватара** с разными характерами (Учёный кот, Саркастичный подросток, Мудрый старец)
- **Стриминг ответов** — сообщения появляются в реальном времени
- **Краткосрочная память** — бот помнит контекст последних 10 сообщений
- **Долгосрочная память** — бот извлекает и запоминает ключевые факты о пользователе
- **Устойчивость фактов** — после `/reset` бот продолжает помнить важные детали

## Стек

- Python 3.10+
- Aiogram 3.x
- SQLAlchemy 2.0 (async) + asyncpg
- PostgreSQL
- OpenRouter API (Google Gemini 3 Flash)
- Alembic (миграции)

## Быстрый запуск

### 1. Клонирование

```bash
git clone <url-репозитория>
cd ai-avatar-bot
```

### 2. Установка зависимостей

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 3. Настройка окружения

```bash
cp .env.example .env
```

Заполните `.env`:

```env
BOT_TOKEN=ваш_токен_бота          # получить у @BotFather
DATABASE_URL=postgresql+asyncpg://bot_user:bot_password@localhost:5432/avatar_bot
OPENROUTER_API_KEY=sk-or-v1-...   # получить на openrouter.ai
```

### 4. База данных

Если нет PostgreSQL, поднимите через Docker:

```bash
docker run -d --name avatar-pg \
  -e POSTGRES_DB=avatar_bot \
  -e POSTGRES_USER=bot_user \
  -e POSTGRES_PASSWORD=bot_password \
  -p 5432:5432 \
  postgres:16-alpine
```

Применить миграции:

```bash
alembic upgrade head
```

### 5. Запуск бота

```bash
python -m bot
```

## Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Начать работу и выбрать аватара |
| `/history` | Показать последние 10 сообщений диалога |
| `/facts` | Показать запомненные факты о вас |
| `/reset` | Очистить историю (факты сохранятся) |
| `/change_avatar` | Сменить аватара |
| `/help` | Справка по командам |

## Конфигурация

Все параметры задаются через переменные окружения в `.env` (см. `.env.example`).

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `BOT_TOKEN` | — | Токен Telegram-бота (обязательно) |
| `DATABASE_URL` | `postgresql+asyncpg://postgres:postgres@localhost:5432/avatar_bot` | URL подключения к PostgreSQL |
| `OPENROUTER_API_KEY` | — | API-ключ OpenRouter (обязательно) |
| `LLM_MODEL` | `google/gemini-3-flash-preview` | Модель LLM |
| `LLM_TEMPERATURE` | `0.7` | Температура генерации |
| `LLM_MAX_TOKENS` | `400` | Максимум токенов в ответе |
| `LLM_TIMEOUT` | `60.0` | Таймаут запроса к LLM (сек) |
| `SHORT_TERM_MESSAGE_LIMIT` | `10` | Кол-во сообщений в краткосрочной памяти |
| `FACT_EXTRACTION_INTERVAL` | `3` | Каждые N сообщений — извлечение фактов |
| `SIMILARITY_THRESHOLD` | `0.75` | Порог дедупликации фактов (0–1) |

## Архитектура

```
bot/
├── handlers/         # Обработчики команд и сообщений
├── services/         # Бизнес-логика (LLM, память, стриминг)
├── database/         # Модели, репозитории, миграции
├── keyboards/        # Inline-клавиатуры
├── middlewares/      # Middleware (сессия БД)
├── states/           # FSM-состояния
└── utils/            # Вспомогательные функции
```

### Модули

- **handlers/** — обработка команд (`/start`, `/history`, `/facts`, `/reset`, `/change_avatar`, `/help`) и текстовых сообщений
- **services/llm.py** — обёртка над OpenRouter API с retry и стримингом
- **services/memory.py** — оркестрация краткосрочной и долгосрочной памяти
- **services/fact_extractor.py** — извлечение фактов из диалога с 4-уровневым парсингом JSON
- **services/streaming.py** — стриминг ответов LLM в Telegram через edit_message
- **database/** — SQLAlchemy 2.0 async модели + репозитории + Alembic миграции

### FSM-состояния

```
/start → [choosing_avatar] → выбор аватара → [chatting] → диалог
                ↑                                    |
                └──── /change_avatar ────────────────┘
```

## Система памяти

### Краткосрочная память
- Хранит последние N сообщений диалога в БД (`dialog_messages`), по умолчанию 10
- Передаётся в LLM как контекст при каждом запросе

### Долгосрочная память
- Каждые N сообщений (по умолчанию 3) бот в фоне извлекает факты о пользователе через дополнительный вызов LLM
- Факты сохраняются в таблицу `memory_facts`
- При каждом обращении факты инъецируются в системный промпт
- Дедупликация через `SequenceMatcher` (порог настраивается)
- Факты переживают `/reset` — бот помнит пользователя даже после очистки диалога

### Парсинг фактов
4-уровневый fallback для надёжного парсинга ответа LLM:
1. Прямой `json.loads()`
2. Regex-поиск JSON-массива в тексте
3. Построчный парсинг (удаление маркеров списка)
4. Весь ответ как один факт

## База данных

4 таблицы:

| Таблица | Назначение |
|---------|-----------|
| `avatars` | Персонажи (имя, промпт, emoji) |
| `users` | Пользователи (telegram ID, текущий аватар) |
| `dialog_messages` | История диалогов |
| `memory_facts` | Долгосрочные факты |

## Демонстрация

### Долгосрочная память после /reset:

1. Пользователь сообщает своё имя в диалоге
2. Бот запоминает факт «Пользователя зовут ...»
3. Пользователь выполняет `/reset` — история диалога очищена
4. Пользователь пишет новое сообщение
5. Бот обращается по имени — факт сохранился в долгосрочной памяти
